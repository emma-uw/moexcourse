# Модуль 4: Каркас торгового робота (Архитектура)

Четвертый модуль курса посвящен созданию надежной архитектуры торгового робота. Мы научимся писать поддерживаемый код, избегая распространенных ошибок начинающих разработчиков.

## Цель

Написать профессиональный код, который легко поддерживать и масштабировать. Создать гибкую структуру, готовую к расширению и модификации.

## Разделы

- [4.1. Почему скрипты ломаются](./4-1-spaghetti.md): Спагетти-код vs Объектно-Ориентированное Программирование (ООП).
- [4.2. Класс Strategy](./4-2-strategy.md): Инкапсуляция логики. Отделяем расчет индикаторов от принятия решений.
- [4.3. Класс DataHandler](./4-3-datahandler.md): Унификация получения данных
- [4.4. Управление состоянием (State Machine)](./4-4-state.md): Состояния робота: WAITING, IN_POSITION, STOPPED. Избегаем открытия дублирующих сделок.

## Результат модуля

Шаблон торгового робота, структура классов Python, готовая к наполнению логикой.
